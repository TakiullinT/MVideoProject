# MVideoProject

Консольное приложение на Kotlin для обработки логов движения товаров на складе (поступление и продажа) с возможностью автоматического покрытия дефицита.
## 📝 Описание проекта

Приложение считывает последовательность операций в формате CSV и рассчитывает финальный остаток товаров на складе. 

**Ключевые особенности реализации:**
* **Учет операций:** Автоматическая обработка поступлений и продаж с сортировкой по приоритету.
* **Определение ранга товара:** ранг товара определяется лексикографической сортировкой по `id` товара. В реализации товар с наименьшим лексикографическим значением (например, `ItemA` < `ItemB`) считается имеющим более высокий приоритет продажи и продаётся первым.
* **Поведение при дефиците:** если продажа превышает суммарный остаток по группе, доступный остаток списывается, а разница записывается как отрицательное значение. При отсутствии товаров в группе дефицит учитывается по ключу `UNKNOWN_ITEM`.
* **Погашение дефицита:** поступления в группе сначала погашают отрицательные балансы `UNKNOWN_ITEM` (если они есть), затем добавляют оставшееся количество на конкретный товар.
* **Надежность:** Использование `TreeMap` для гарантированной сортировки результатов и эффективный парсинг с защитой от некорректных данных.
* **Неправильные/пустые строки:** пустые строки игнорируются; строки с неверным количеством полей или нечисловыми количествами логируются предупреждением и пропускаются.

## ⚙️ Требования к окружению

* **Java Development Kit (JDK) 22** (или выше)
* Установленный локально сборщик Gradle **не требуется** — проект содержит настроенный Gradle Wrapper (`gradlew`).

## 🛠 Сборка и запуск

Проект использует **Gradle Wrapper**, поэтому установка Gradle не требуется. Используйте терминал в корне проекта:

* **Сборка проекта:**
```bash
./gradlew build

```


* **Запуск:**
```bash
./gradlew run --args="input.csv output.csv"

```


* **Запуск тестов с детальным выводом:**
```bash
./gradlew test --console=plain --info

```

## 📊 Примеры данных (CSV)

Программа принимает на вход строки формата CSV (разделитель `;`).

**Входные данные (Input):**

```csv
Group1;Item1;10
Group2;ItemX;5
Group1;4
Group1;Item2;5
Group1;7

```

**Выходные данные (Output):**
Результат автоматически сортируется по алфавиту (группа, затем товар) с учетом покрытого дефицита:

```csv
Group1;Item1;0
Group1;Item2;4
Group2;ItemX;5

```
